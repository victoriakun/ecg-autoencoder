# External-database validation

Generated by `tools/external_eval.py` on top of `models/mitbih_autoencoder_best.pt`
using the real-time pipeline path (Option C: warmup-derived global
normalization).

## Datasets used

| Database | PhysioNet name | Native fs | Records used | Min/record | Total scored |
|---|---|---|---|---|---|
| **In-distribution** | mitdb (MIT-BIH ARR) | 360 Hz | training-set records | full | (see `clinical_proof.pdf`) |
| **External — healthy** | nsrdb (MIT-BIH NSR) | 128 Hz | 16265, 16272 | 30 min | 7,166 windows |
| **External — AF** | afdb (MIT-BIH AF) | 250 Hz | 04015, 04043, 04048 | 30 min | 10,749 windows |

External records are resampled to 360 Hz before being fed into the model.

## Results at a glance

| Setting | Threshold | Sensitivity (to anomaly / AF) | Specificity (on normal / non-AF) | False-positive rate |
|---|---|---|---|---|
| **MIT-BIH ARR** (training distribution) | 0.043 (F1-optimal) | **91 %** | **92 %** | 8 % |
| **MIT-BIH NSRDB** (healthy, different cohort) | 0.043 (fixed) | n/a — all healthy | **0.7 %** | **99.3 %** |
| **MIT-BIH AFDB** (AF episodes) — fixed | 0.043 (fixed) | 99.4 % | 1.4 % | **98.6 %** |
| **MIT-BIH AFDB** — per-record adaptive p95 | dynamic | 15.1 % | 96.3 % | 3.7 % |

## Honest reading

**The model does not generalize to NSRDB or AFDB out of the box.** Median
residual on these external records (~0.35) is roughly **8× higher** than
the in-distribution F1-optimal threshold (0.043). Three confounds at
once:

1. **Sampling rate mismatch** — model trained at 360 Hz, NSRDB at 128 Hz,
   AFDB at 250 Hz. Resampling preserves timing but not spectral shape.
2. **Patient cohort drift** — MIT-BIH ARR has older outpatients with
   known arrhythmias; NSRDB is healthy volunteers; AFDB is paroxysmal
   AF patients.
3. **Recording equipment / lead placement** differs across all three.

Even **adaptive (per-record) thresholding doesn't recover AF separation
on AFDB** (15 % sensitivity at 96 % specificity). The model's residuals
on AFDB are roughly equally high for AF and non-AF windows — meaning the
representation it learned on MIT-BIH ARR doesn't transfer to AFDB's
signal characteristics, regardless of threshold choice.

## What this means for the thesis

This is the kind of finding a defence committee will appreciate
*because* it's reported honestly. Three takeaways:

1. **In-distribution performance is real** — ROC-AUC 0.97, F1 0.85 on
   MIT-BIH ARR is reproducible and well-documented.
2. **External generalization is a clear limitation** — should be
   explicit in the thesis Limitations section, with these numbers
   cited.
3. **Per-cohort retraining or domain-adaptation** is the obvious
   future-work direction. PTB-XL (12-lead, 21k modern recordings) is
   the most realistic next benchmark.

Use this in the Discussion chapter to argue that *any* serious clinical
deployment would need re-calibration on the target hospital's own data
before going live — which is a feature, not a bug, of an unsupervised
autoencoder approach.

## Reproducing

```bash
PYTHONPATH=. python tools/external_eval.py --db nsrdb \
    --records 16265,16272 --max-seconds 1800 \
    --out-json docs/external_eval/nsrdb.json

PYTHONPATH=. python tools/external_eval.py --db afdb \
    --records 04015,04043,04048 --max-seconds 1800 \
    --out-json docs/external_eval/afdb.json
```

JSON outputs in this directory contain full per-record stats.
